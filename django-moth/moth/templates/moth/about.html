This software is the evolution of the old, Apache+PHP based moth software which helped us test w3af for many years. I decided to move towards Django because:

  • "python manage.py runserver" allows us to run a test server anywhere, more specifically in the continuous integration system where w3af's unit-tests are run
  • Easy to implement any vulnerability in Python
  • A modern development framework, which I'll also use for delivering web application security trainings.

The rewrite was mainly driven by the need of running w3af's tests in a continuous integration system, without any external dependency (such as the old moth vm).

This software should never be used in a production environment.
